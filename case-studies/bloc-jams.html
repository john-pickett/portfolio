Bloc Jams Case Study

Summary

There's nothing better than getting to jam out with your favorite music, and you definitely need to have your music with you on the go too! Bloc Jams is a handy, mobile-friendly music player that you can take with you anywhere!

Bloc Jams has several different views: the landing page, the collection view, and the album view. The landing page is the first page that a user would see. It has a splashy image up top, and several benefit-oriented bulletpoints for the user. The collection page houses all of the user's music, sorted by album. The album page has the information for the selected album, all of the songs, as well as the music player itself.

Challenges

The biggest challenge in making Bloc Jams was the album page, and the sheet number of functions that it needed to have in order for everything to work correctly. These functions included:

  • list the album's songs
  • show the album information (artist, year, etc)
  • allow the user to select a song to play
  • allow the user to pause the song and/or select a new song
  • show the correct time information for the playing song
  • allow the user to change the song volume

Angular proved to be a great framework for building Bloc Jams, because it allowed me to create a song player factory to house these functions, and then access them from different parts of the album player page. This allowed me to keep my code neat and orderly, which is always a goal worth pursuing.

Conclusion

Building Bloc Jams was a great learning experience, and I enjoyed it thoroughly. It would be fun to add in some new features, such as the ability to choose a song randomly, or being able to listen to your entire collection of music from one page. Perhaps in Bloc Jams 2!
