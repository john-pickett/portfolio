Bloc Chat Case Study

Summary

Everyone needs a way to stay in touch with friends and a chat room is a great way to do it! Plus, you might meet some new friends who share your interests.

Overview of the App

Bloc Chat was made with AngularJS on the front end, and connected to a Firebase database on the back end. The chat room names and the messages in each chat room are stored in the database, and pulled when the user selects a new chat room to join.

The user can also choose their username, and it is stored locally in a cookie. The next time the user comes to the chat rooms, their username is populated automatically. Users can also reset their username if needed.

Challenges

One unique challenge that I ran into in this project was when I was creating the controller function to allow the user to create a new username. I wanted each user to have to provide a valid username so that their messages would be properly associated with their username. I ended up validating the username like so:

chatapp.newUser = function(){
 var modalInstance = $uibModal.open({
  templateUrl: '/templates/user-modal.html',
  controller: function($scope, $uibModalInstance) {
   $scope.create = function() {
    if ($scope.newUser !== undefined && $scope.newUser != ""){
      $uibModalInstance.close($scope.newUser);
    } else {
      alert("Error: Please provide a valid username");
    }
   };
  },
   size: 'md',
});

When the user clicks the button to reset their username, a modal pops up allowing them to input a new name. If they don't enter a name into the input field, they get an alert saying "Please enter a valid username", and they cannot exit the modal until they do so.

Conclusion

The chat app was a lot of fun to make, and Angular was a great resource to use on this project. Building this as an SPA and using ui-view makes switching from room to room and displaying different messages quick and easy for the user.
